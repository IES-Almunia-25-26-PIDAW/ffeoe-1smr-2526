## Objetivos

Al finalizar esta actividad el alumno será capaz de:

- Diagnosticar averías hardware mediante metodología sistemática.
- Sustituir o ampliar componentes siguiendo procedimientos correctos.
- Documentar una incidencia desde su apertura hasta su cierre.
- Comunicarse de forma eficaz con el usuario final para recoger información del problema.

## Materiales necesarios

- Sistema de tickets de incidencias de la empresa (si existe) o ficha de incidencia en papel.
- Herramientas de diagnóstico hardware.
- Componentes de repuesto o acceso al almacén de la empresa.
- Herramientas físicas (destornilladores, pulsera antiestática).

## Desarrollo

### 1. Registro de la incidencia

Atiende o solicita a tu tutor/a que te asigne al menos **2 incidencias hardware reales**
durante el periodo de FFEOE. Para cada incidencia, cumplimenta la ficha de registro:

|---|---|
| Nº de ticket / referencia | |
| Fecha y hora de apertura | |
| Usuario o departamento afectado | |
| Descripción del problema (en palabras del usuario) | |
| Prioridad (Baja / Media / Alta / Crítica) | |

### 2. Diagnóstico

Aplica la metodología de diagnóstico sistemático:

1. **Reproducción del problema**: intenta reproducir el fallo en condiciones controladas.
2. **Recogida de información**: ¿Cuándo empezó? ¿Es continuo o intermitente? ¿Hay mensajes de error?
3. **Comprobaciones básicas**: conexiones, alimentación, indicadores LED.
4. **Pruebas de componentes**: sustitución por intercambio (swap) o uso de herramientas de diagnóstico.
5. **Identificación de la causa raíz**.

### 3. Resolución

Documenta la solución aplicada:

- Componente sustituido o reparado.
- Procedimiento seguido (incluyendo medidas de seguridad ESD).
- Verificación de que el problema está resuelto.
- Pruebas realizadas con el usuario final para confirmar la solución.

## Documentación a entregar

- Ficha completa de cada incidencia (apertura, diagnóstico y cierre).
- Reflexión personal (media página) sobre el proceso de comunicación con el usuario.
