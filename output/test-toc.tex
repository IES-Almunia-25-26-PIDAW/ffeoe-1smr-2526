\textbf{Parte 1 --- Localización de elementos en el plano}

En el plano del centro, toma nota de la ubicación de:

\begin{itemize}
\tightlist
\item
  El armario principal
\item
  Los armarios de distribución, si los hay
\item
  Cada una de las rosetas, indicando la etiqueta de las tomas que
  contiene
\item
  Recorridos del cableado (habrá partes no visibles; intuye por dónde
  van)
\end{itemize}

\textbf{Parte 2 --- Contenido del armario principal y los armarios de
distribución}

De cada armario:

\begin{itemize}
\tightlist
\item
  Toma nota de los elementos que hay. Haz una lista indicando para cada
  uno qué función tiene.
\item
  Haz un esquema gráfico de esos elementos dentro del armario.
\end{itemize}

\textbf{Parte 3 --- Identificación de todos los puertos de paneles de
parcheo y rosetas}

\begin{itemize}
\tightlist
\item
  Comprueba que todos los puertos de los paneles de parcheo tienen una
  etiqueta identificable y que sabes dónde termina (en qué roseta).
\item
  Comprueba que todas las rosetas están bien identificadas y sabes dónde
  terminan (en qué armario, panel y puerto).
\item
  Si algún puerto o toma no está identificado, investiga para intentar
  averiguarlo.
\item
  Completa en la tabla las columnas: Armario / Panel / Puerto, Etiqueta,
  Sala donde termina, Etiqueta en roseta.
\item
  Si es posible desconectar temporalmente lo que esté conectado,
  verifica la conexión con un tester y anota el resultado.
\end{itemize}

\textbf{Parte 4 --- Esquema lógico de la red}

\begin{itemize}
\tightlist
\item
  Identifica todos los switches de la instalación y representa un
  esquema lógico de su conexión. Si algún switch está fuera de un
  armario, indícalo.
\item
  Localiza en la documentación del armario las VLANs definidas e
  identifica a qué VLAN está asignado cada puerto.
\end{itemize}

\textbf{Parte 5 --- Estado de conectividad de la red}

\begin{itemize}
\tightlist
\item
  Para cada puerto del panel de parcheo, comprueba si tiene un
  latiguillo hasta el switch correspondiente. Completa las columnas:
  Switch / Puerto, VLAN, Conexión / Velocidad.
\item
  Para cada puerto del panel de parcheo, intenta identificar qué
  elemento hay conectado e indícalo en la tabla.
\end{itemize}

red}{AF108 - Diagnóstico de incidencias y/o chequeo completo de la red}}\label{af108---diagnuxf3stico-de-incidencias-yo-chequeo-completo-de-la-red}

\begin{itemize}
\tightlist
\item
  Resultado de aprendizaje: \texttt{RA5}
\item

\paragraph{Descripción de la tarea:}\label{descripciuxf3n-de-la-tarea-1}

\emph{(Se realizará durante la semana 2 y 3)}

Para esta actividad necesitarás:

\begin{itemize}
\tightlist
\item
  Haber reconocido la red en la actividad anterior.
\item
  Herramientas para el diagnóstico de averías: tester, ordenadores, etc.
\end{itemize}

\textbf{Producto final:} Partes técnicos de incidencias detectadas según
modelo proporcionado.

\textbf{Opción primera: si en el CEIP (cliente) nos han indicado
incidencias detectadas}

Para cada incidencia:

\begin{itemize}
\tightlist
\item
  Hacer las pruebas pertinentes para determinar si se trata de un fallo
  hardware o software, así como el elemento que provoca el fallo.
\item
  En el parte de incidencias indicar las pruebas realizadas y la
  conclusión que se extrae de cada una de ellas.
\item
  Dar un diagnóstico para la incidencia. En este punto no se
  propone/aplica una solución; se hará en el ejercicio siguiente.
\end{itemize}

\textbf{Opción segunda: si en el CEIP (cliente) no nos han indicado
ninguna incidencia o ya las hemos tratado todas}

\begin{itemize}
\tightlist
\item
  Hacer una revisión física con un tester de cada una de las tomas y
  anotar las incidencias detectadas, cada una en un parte técnico de
  incidencias.
\item
  Hacer pruebas de conectividad y velocidad en cada ordenador y anotar
  las incidencias detectadas, cada una en un parte técnico de
  incidencias.
\end{itemize}

\paragraph{Entrega:}\label{entrega-1}

\begin{itemize}
\tightlist
\item
  Para el profesorado: escaneos de los partes técnicos con las
  incidencias comunicadas o detectadas en la revisión.
\item
  Para el CEIP: escaneos de los partes técnicos con las incidencias
  comunicadas o detectadas en la revisión.
\end{itemize}
